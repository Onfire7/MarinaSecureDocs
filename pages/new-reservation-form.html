Reservations · Page Specification

New Reservation Form

Book a reservable Location or Asset for a contact — gated by manage_reservations.

Purpose

Reached from Reservation calendar/list, or directly from a reservable Location/Asset's detail page. The target picker only ever offers Locations with reservation_enabled on and Assets with the same — nothing else is selectable, which is what keeps a mixed-use Dock's leased Slips out of this flow entirely.

Data shown / form fields

FieldTypeNotes
TargetSelect — reservable Locations and Assets onlyPre-filled when opened from a specific Location/Asset's context.
ContactSearch/select, or create newSame contact picker used elsewhere (see Owner/Contact list).
Expected check-in / check-outDate
Rate / depositCurrencyShown only if the selected target's reservation_visibility is Public; the field group itself appears/disappears as the target selection changes.

Actions

ActionGatingBehavior
Submitmanage_reservationsCreates the Reservation with status Requested (or Confirmed, if the marina's workflow skips the request step — an admin-configurable default, not modeled as a separate field). Routes to Reservation detail on success.
CancelNoneCloses without creating a reservation.

States & edge cases

  • Target not reservation-enabled — never appears in the picker; there's no way to attempt booking a non-reservable Location or Asset from here.
  • Overlapping dates on the same target — governed by MarinaSettings.allow_overlapping_reservations (off by default, see Marina Settings): with the default off, submitting is blocked outright with an error naming the conflicting reservation; a marina that wants a waitlist/overbooking model can turn the setting on, at which point overlapping dates are simply permitted with no confirmation friction.
  • No manage_reservations — this form is unreachable; every entry point to it is gated the same way.
  • Internal-only target — rate/deposit fields never appear, and the resulting reservation has no billing surface anywhere downstream.

Mobile vs. desktop

Same form on both; the target picker on desktop can optionally show the schematic map from Reservation calendar/list for point-and-click selection, since there's room for it — mobile sticks to a search/select list.
