Reservations · Page Specification

Reservation Calendar / List

Every reservation across every reservable Location and Asset — cabins, RV sites, slips, jump packs, whatever a marina has enabled — viewed as a calendar/list, or as a schematic map with color-coded slip rectangles.

Purpose

Reached from the Reservations top-level nav item. One screen serves every reservable target type uniformly: Reservation.target is a Location or an Asset, and this list doesn't care which — a cabin booking and a jump-pack checkout-by-reservation look the same here beyond their target's type label. A location only ever appears as a possible target if its own reservation_enabled is on, which is what makes a mixed-use Dock work (some Slips bookable here, others tied to a Lease and simply absent from this screen) — see Data Model — Location.

Map view

A schematic view built on MarinaMap + LocationMapPlacement (see Data Model): the same plotted rectangles used by Location list / map view's own schematic mode, here color-coded by reservation status instead of occupancy — e.g. green for vacant/available, amber for an upcoming reservation, red for currently checked in — and each rectangle labeled with the location's name. Starts on the marina-wide master map; drilling into a parent Location (where one exists) switches to that Location's own detail map, zoomed in on just its children. Asset-target reservations (a jump pack, not a Location) have no rectangle to plot and so only ever appear in the list/calendar view, not on the map.

Actions

ActionGatingBehavior
Toggle List / Calendar / Map viewNoneSame underlying filtered result set, three presentations.

States & edge cases

  • No dedicated view permission — same as tickets and reservations elsewhere in the app, browsing this screen is unrestricted; only creating/editing requires manage_reservations, per Permissions.
  • Location not yet plotted on any map — still fully usable in list/calendar view; simply absent from the map view until an admin plots it, per Admin → Location Types & Locations setup.
  • Upcoming reservation vs. current status — a target's map rectangle color reflects both: its current occupancy status and, distinctly, an "upcoming" indicator if a reservation is confirmed but not yet active — matching the "displayed alongside, not replacing" rule from Data Model — Reservation.
  • Mixed-use dock — on the map, a Dock's detail map shows only its reservable children; Slips with reservation_enabled off (leased long-term) don't get a rectangle here at all, since they're not part of this screen's dataset.

Mobile vs. desktop

Mobile: list view by default, calendar and map as full-screen alternate modes reached via a toggle. Desktop: calendar or map can sit alongside a filterable list, similar to the split used on Location list.
