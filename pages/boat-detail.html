Boat Detail

Everything about one boat — owners in succession order, authorized users, current slip, and its lease if it has one.

Purpose

Reached from Boat list, Location detail (a Slip's current boat), or a Reservation/Ticket/Incident/Note pointing at it.

Data shown

FieldSourceNotes
Name, description, length/make/modelBoat fields
Registration numberBoat.registration_number
OwnersBoat.owners (ordered)Requires view_owner for names; view_contact additionally for phone/email. Order shown explicitly as succession — who to contact first.
Authorized usersBoat.authorized_usersSame gating as owners.
Current slipBoat.current_slipLinks to Location detail; nullable.
LeaseLease where location = current_slipRequires view_lease; omitted entirely without it.
Notes, incidents, ticketsattached_to = this BoatSame gating as everywhere else these appear (incidents require view_incidents; tickets and notes unrestricted).

Actions

ActionGatingBehavior
Edit boat detailsedit_owner_contactSame permission as boat creation on Boat list.
Reassign current slipedit_owner_contactUpdates Boat.current_slip; the move itself is preserved via the Activity Log rather than a slip-history table, per Data Model.
Add a noteUnrestrictedOpens New note dialog, pre-attached to this Boat.
Log an incidentcreate_incidentsOpens New incident form, pre-attached.
Create a ticketUnrestrictedOpens New ticket form, pre-attached.

States & edge cases

  • No view_owner — owners and authorized users sections are both omitted entirely, matching the gating used on Location detail.
  • No current slip — shown as an explicit "unassigned" state rather than an empty field, since a boat with no slip is a meaningfully different situation from one whose slip just isn't loaded yet.
  • No lease, or no view_lease — the lease section is omitted either way; the UI doesn't distinguish "no lease exists" from "you can't see it."

Mobile vs. desktop

Mobile: sections stack vertically, collapsible. Desktop: sections visible together in a wider layout, matching the pattern used on Location detail.
