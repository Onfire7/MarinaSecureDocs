Location Types & Locations Setup

Define the marina's location hierarchy, lay out its actual docks/slips/buildings, configure checkpoints, and — the piece deferred since Reservations — upload maps and plot locations onto them.

Purpose

Reached from Admin Home, gated by manage_locations. Three nested concerns live here: the type system (what kinds of location exist and how they nest), the actual location instances (the marina's real docks and slips), and now the map-plotting layer that backs the schematic map views on Location list and Reservation calendar/list.

Map upload & plotting

Per the note carried since Reservations, backed by MarinaMap and LocationMapPlacement in Data Model:

FieldSourceNotes
Map name & imageMarinaMap.name / imageUploaded layout/diagram — a schematic, not a real-world map.
ScopeMarinaMap.scopeNull for the one marina-wide master map; set to a parent Location for a detail map showing just that parent's children.
Plotted rectanglesLocationMapPlacement (location, map, rect)Drag-and-place a rectangle per Location onto the selected map's image; a Location can be plotted on more than one map (e.g. small on the master map, precise on its parent's detail map) with independent coordinates on each.

Actions

ActionGatingBehavior
Create / edit a LocationTypemanage_locations
Create / edit a Locationmanage_locationsIncluding the reservation toggle and post-reservation status.
Add / edit a Checkpointmanage_locationsGUID URL is generated automatically on creation, not user-entered; GPS radius override defaults to the marina-wide default from Marina Settings.
Upload a mapmanage_locationsCreates a MarinaMap; only one master map (scope = null) is expected, though the data model doesn't hard-enforce that — the implementing team should confirm whether to block creating a second master map or leave it a marina-configuration convention, matching how Lease detail handles the reservation/lease split.
Plot a Location on a mapmanage_locationsDrag-and-place (or click-and-resize) a rectangle; creates/updates the corresponding LocationMapPlacement.
Unplot a Location from a mapmanage_locationsRemoves just that one LocationMapPlacement — a Location plotted on multiple maps stays on the others.

States & edge cases

  • Deleting a LocationType with existing Locations — should be blocked or require reassigning those Locations first; deleting the type out from under them would leave orphaned records.
  • Location not yet plotted on any map — fine; it simply doesn't appear in the schematic map view mode until plotted, per Location list.
  • Detail map for a parent with no children yet — allowed to create ahead of time; just has nothing to plot until child Locations exist.
  • Reservation toggle turned off while a Reservation is active — the existing Reservation isn't retroactively invalidated; only future booking through New reservation form is affected, since that form's target picker only offers currently-enabled locations.

Mobile vs. desktop

Desktop is the primary surface, especially for map plotting, which needs precise drag-and-place interaction and screen width to see the whole layout at once. Mobile supports viewing and light field edits (status labels, a checkpoint's GPS override) but not the map-plotting workflow.
