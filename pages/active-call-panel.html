Active Call Panel / Pop-Up

Appears automatically whenever a call is ringing or in progress — a right-side panel on desktop, a pop-up on mobile — surfacing everything useful about the caller before anyone has to ask.

Purpose

Per Role Workflows — Handling calls, the goal is that whoever answers already has context: who's calling, what they might be calling about, and what's already been discussed with them. Appearance is driven by the call ringing to this user's device, not by a permission — place_calls gates initiating a call, not receiving one, so this panel can surface for anyone a call is routed to.

Data shown

FieldSourceNotes
Call stateCall.direction, line, started_at, live durationDuration ticks live while the call is in progress.
Matched contactCall.contact → ContactName always shown; phone/email require view_contact, matching the gating used everywhere else a Contact appears.
Active reservationsReservation.contact = this Contact, status Confirmed/Checked InExpected/actual check-in-out and target, so an office user can answer a booking question without switching screens.
Related slip / boat infoBoat.owners / Boat.current_slip for this ContactGated by view_owner as elsewhere.
Prior call notesCallNote for this Contact's past CallsRecent history, not just this call's notes.
Recording / transcriptCall.recording_url / Call.transcriptOnly shown if MarinaSettings.call_recording_enabled / call_transcription_enabled; populated by the Twilio Function once available, not at call start.

Actions

ActionGatingBehavior
Add a noteUnrestrictedCreates a CallNote live, during or after the call.
End the callplace_callsHangs up via the Twilio Function; the Function updates Call.status/duration on the resulting webhook, per the Twilio-source-of-truth rule in Architecture.
Dismiss the panelNoneCloses the panel without ending the call (e.g. a call already handled by someone else).

States & edge cases

  • Nameless contact — no match found for the incoming number; the panel shows a prompt to enter a name, then surfaces similar existing Contacts for a merge, per Role Workflows and Nameless-contact merge.
  • Recording/transcription disabled for the marina — that section is omitted entirely, not shown empty.
  • No active reservation or slip/boat info — those sections are omitted rather than shown as empty placeholders.
  • Call ends — the panel stays open briefly for a final note before collapsing, rather than disappearing the instant the call drops.

Mobile vs. desktop

Desktop: a persistent right-side panel, so the rest of the app stays usable underneath it. Mobile: a pop-up (bottom sheet or modal) since there's no room for a persistent sidebar — matches the exact split described in Role Workflows.
