Comms · Page Specification

New Chat Room / Invite Screen

Create a topic-based chat room and choose who's in it — specific users, whole roles, or both.

Purpose

Reached from Comms Home. Room creation itself is unrestricted — any user can start one, per Role Workflows — since chat is a coordination tool, not a data-access surface that needs gating.

Data shown / form fields

FieldTypeNotes
TitleText, requiredChatRoom.title
TopicText, optionalChatRoom.topic
Invite usersMulti-selectChatRoom.invited_users
Invite rolesMulti-selectChatRoom.invited_roles — a live reference, so anyone later granted that role gains access automatically (see Chat Room).

Actions

ActionGatingBehavior
CreateUnrestricted — any userCreates the ChatRoom with created_by = self, then navigates into Chat Room. Selecting zero invitees is allowed — a valid personal scratch room, not an error.
CancelNoneCloses without creating a room.

States & edge cases

  • No invitees selected — still creates the room; the creator can always add invitees later from within Chat Room.
  • Inviting a role with no current holders — allowed; the room simply has no participants from that role yet until someone is granted it.
  • Duplicate-sounding titles — not deduplicated or blocked; titles are freeform labels, not identifiers.

Mobile vs. desktop

Same form-style screen on both; no meaningful layout divergence beyond the app's general responsive treatment.
