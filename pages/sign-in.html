Sign In

The entry point for any user with no active local session on this device — authenticates through Clerk, which InstantDB is configured to trust as its auth provider.

Purpose

Every deployment is a single marina's own subdomain (see Architecture), so there's no marina picker here — this screen only ever authenticates a User against the one Clerk application (and one InstantDB instance) that subdomain already points to. There is no self-serve account creation: a user must already exist, created via Admin → Users by someone holding manage_users, which provisions them in both Clerk and as an InstantDB User record (clerk_user_id linking the two — see Data Model).

Data shown

FieldSourceNotes
Marina brandingMarinaSettingsName/logo, so the correct property is obvious before entering credentials.
Identifier fieldClerk sign-in formRendered via Clerk's own hosted/embedded sign-in component; identifier and challenge (email/password, magic link, or OTP) are configured per marina in that marina's Clerk dashboard, not hand-built here.

Actions

ActionGatingBehavior
Sign inNone (this is the gate)Handled by Clerk; on success, Clerk issues a session, InstantDB verifies it via its Clerk integration, and the app routes to the Dashboard.
Resend / forgot credentialNoneStandard Clerk recovery flow for whichever challenge method is configured.
Switch to a different userShown only when this device already has at least one signed-in Clerk sessionRoutes to User Switch, backed by Clerk's multi-session support, instead of a fresh sign-in.

States & edge cases

  • First sign-in on a device requires connectivity — there's no local session yet for InstantDB to validate against offline. Once signed in, the session persists locally and later app use doesn't require this screen again.
  • Deactivated user (User.active = false) — fails with the same generic error as a wrong credential; the UI does not reveal that the account exists but is deactivated.
  • Already has a valid local session — this screen is skipped entirely on launch; the app opens straight to the Dashboard.

Mobile vs. desktop

Identical single-column form on both. No layout divergence — this is the one screen where a native-app feel and a webapp feel don't need to differ.
