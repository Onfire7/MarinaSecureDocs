Comms · Page Specification

Comms Home

The landing point for the Comms top-level nav item — recent calls, texts, and chat rooms in one place, and the entry point for starting any of the three.

Purpose

An overview, not a permission gate: viewing call and SMS history has no dedicated permission in the fixed catalog (same reasoning as tickets and reservations — see Permissions), only initiating a call or SMS requires place_calls. Chat rooms are the one section here with real visibility scoping — a user only sees rooms they're invited to, unless they hold view_all_chats.

Data shown

SectionSourceNotes
Recent callsCall, most recent firstDirection, matched Contact (gated by view_owner/view_contact the same as everywhere a Contact appears), line, time, duration or missed flag.
Recent textsSMSThread, ordered by last_message_atContact, line, last message preview, unread indicator.
Chat roomsChatRoom where the user is created_by, in invited_users, or holds a role in invited_roles — or all rooms, with view_all_chatsTitle, most recent activity.

Actions

ActionGatingBehavior
Start a callplace_callsOpens New call dialog.
Start a textplace_callsOpens New SMS dialog.
Start a chat roomUnrestricted — any userOpens New chat room / invite screen.
Open a call entryNoneExpands inline — matched contact, notes, recording/transcript if enabled — rather than a separate call-detail page; a live call instead surfaces via Active call panel.
Open a text threadNoneNavigates to SMS thread; marks it read.
Open a chat roomMust be invited, or hold view_all_chatsNavigates to Chat room.

States & edge cases

  • No place_calls — the start-call and start-text actions are hidden entirely, but existing call/text history is still visible; a marina can have staff who need call context (e.g. reviewing a call log) without being able to place calls themselves.
  • Empty sections — each of the three sections has its own empty state rather than hiding the whole section.
  • Not invited to any chat room and no view_all_chats — the chat section shows only the "start a room" action with nothing listed, rather than being hidden outright.

Mobile vs. desktop

Mobile: calls, texts, and chat rooms as three switchable tabs to keep each list full-width and scannable. Desktop: the three sections can sit side by side in columns, since there's room to scan all of them without switching.
