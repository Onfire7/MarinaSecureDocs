Comms · Page Specification

Comms Home

The landing point for the Comms top-level nav item — recent calls, texts, and chat rooms in one place, and the entry point for starting any of the three.

Purpose

Three independently-gated sections sharing one screen: view_calls controls the calls section, view_sms controls the texts section, and chat rooms split into "My chat rooms" (participant, always visible) and "Other chat rooms" (visible only with view_all_chats, and only readable until the user joins — see Chat Room). None of these overlap with place_calls, which gates only initiating a call or SMS — a marina can have staff who see call/text history without being able to place calls, or who can place calls but can't browse past ones, per Permissions.

States & edge cases

  • No view_calls — the calls section (and its tab, on mobile) is omitted entirely, not shown empty; starting a new call is still possible with place_calls even without it, since the two are independent.
  • No view_sms — same treatment for the texts section.
  • No place_calls — the start-call and start-text actions are hidden, but existing history stays visible to anyone holding view_calls/view_sms; a marina can have staff who need call/text context without being able to place calls themselves.
  • Empty sections (as opposed to gated-off ones) — each has its own empty state rather than hiding the whole section.
  • No view_all_chats — "Other chat rooms" is omitted entirely; the user sees only "My chat rooms," which shows just the "start a room" action if that's also empty.
  • Joining a room from "Other" — once the user performs the Join action on Chat Room, that room moves to "My chat rooms" on next load, since participant status is what determines the split.

Mobile vs. desktop

Mobile: calls, texts, and chat rooms as three switchable tabs to keep each list full-width and scannable; within the chat tab, "My chat rooms" and "Other chat rooms" are stacked sections rather than a further sub-tab. Desktop: the three top-level sections can sit side by side in columns, with the chat column showing both "My" and "Other" sections at once.
