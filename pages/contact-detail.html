Owners & Contacts · Page Specification

Owner / Contact Detail

Everything tied to one person — contact info, the boats they own or are authorized on, leases, and anything attached to them.

Purpose

Reached from Owner/Contact list, a Boat's owners/authorized users, a Lease's lessees, or an inbound Call/SMS's matched contact. Opening a nameless contact (name = null) routes straight into the name prompt & merge dialog instead of this page — this page assumes a named contact.

Data shown

FieldSourceNotes
NameContact.nameRequires view_owner.
Phone / emailContact.phone / emailRequires view_contact.
Boats ownedBoat.owners containing this ContactShown in succession order where relevant; links to Boat detail.
Boats authorized onBoat.authorized_users containing this Contact
LeasesLease.lessees containing this ContactRequires view_lease; omitted entirely without it.
Notes, incidents, ticketsattached_to = this ContactStandard gating (incidents require view_incidents; notes and tickets unrestricted).
Call / SMS historyCall.contact / SMSThread.contact = this ContactRequires view_calls / view_sms respectively, per Permissions.

Actions

ActionGatingBehavior
Edit contact infoedit_owner_contact
Merge with another contactedit_owner_contactManually triggered version of the automatic nameless-contact merge — sets this Contact's (or the other one's) merged_into; anywhere a Contact is displayed, the app resolves through merged_into to the canonical record rather than rewriting every Call/SMSThread/Boat reference.
Call / textplace_callsOpens New call dialog or New SMS dialog pre-filled with this contact.
Add a noteUnrestrictedOpens New note dialog, pre-attached to this Contact.
Log an incidentcreate_incidentsOpens New incident form, pre-attached.
Create a ticketUnrestrictedOpens New ticket form, pre-attached.

States & edge cases

  • No view_contact — name and boat/lease relationships still show (given view_owner); phone/email are omitted, and consequently the call/text actions have nothing to pre-fill from unless the user separately knows the number.
  • Merged contact viewed directly — reroutes to the canonical target's detail page rather than showing a stub for the merged-away record.
  • No linked boats or leases — those sections are simply omitted, same as elsewhere in the app.

Mobile vs. desktop

Mobile: sections stack vertically, collapsible. Desktop: sections visible together in a wider layout, matching Boat detail's pattern.
