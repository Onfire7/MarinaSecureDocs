Tours Setup

Define which checkpoints make up a security tour, and how the app sequences them.

Purpose

Reached from Admin Home, gated by manage_locations — reused rather than introducing a dedicated Tours permission, since a Tour is fundamentally a composition of Checkpoints, which already fall under this permission's scope.

Data shown

FieldSourceNotes
NameTour.name
ModeTour.modeLinear / Freeform / Randomized — see Role Workflows for the guard-facing behavior each implies.
CheckpointsTour.checkpointsOrdered for Linear mode; an unordered set for Freeform/Randomized.

Actions

ActionGatingBehavior
Create / edit a Tourmanage_locations
Reorder checkpointsmanage_locationsOnly meaningful for Linear mode; drag-to-reorder.
Add / remove a checkpointmanage_locationsOnly existing Checkpoints (defined in Location Types & Locations setup) can be added — this screen doesn't create new checkpoints itself.
Change modemanage_locationsSwitching to/from Linear discards or requires establishing an order, since Freeform/Randomized don't carry one.

States & edge cases

  • Checkpoint belongs to multiple Tours — allowed; Checkpoint.tours is a list, so the same physical checkpoint can participate in more than one tour definition.
  • Empty Tour — a Tour with zero checkpoints saves without any warning; it's simply not meaningfully assignable to a shift until checkpoints are added, no different from saving any other config screen mid-setup.
  • Randomized mode with very few checkpoints — the unpredictability the mode is meant to provide degrades with a small set; not blocked, just a natural limitation of the marina's own layout.

Mobile vs. desktop

Desktop is the primary surface for reordering a Linear tour's checkpoint sequence. Mobile supports viewing and simple field edits (name, mode) but not drag-reordering a long checkpoint list.
