Reservation Check-In / Check-Out Dialog

Marks a reservation's actual arrival or departure — and, as a side effect, moves the target into whatever status that implies.

Purpose

Reached from Reservation detail. Deliberately a separate dialog rather than an inline date edit, because setting actual_checkin/actual_checkout has a real side effect on the target's status that a plain field edit shouldn't casually trigger — see Data Model — Reservation.

Data shown

FieldSourceNotes
Reservation summaryReservation.contact / target / expected datesRead-only context for the action being taken.
Actual date/timeSet on submitDefaults to now; editable for backfilling a check-in/out that happened earlier.

Actions

ActionGatingBehavior
Check inmanage_reservationsSets actual_checkin and status = Checked In. The target's own status updates to reflect occupancy (e.g. a Slip → Occupied) — this is the one moment an upcoming reservation actually changes the target's status, per Data Model.
Check outmanage_reservationsSets actual_checkout and status = Checked Out. The target moves to its configured post_reservation_status (e.g. Cabin → "Needs Cleaning," Jump Pack → "Needs Charging," via Asset.post_return_status for asset targets).
CancelNoneCloses without changing anything.

States & edge cases

  • Check-in before expected date — allowed; for a Public reservation this is what early_checkin represents, recorded alongside rather than blocking the action.
  • Check-out after expected date — same treatment via late_checkout, Public reservations only.
  • Checking in an Asset target — identical flow; "the target's status" means Asset.current_status for an Asset, same mechanics as a Location's status.
  • Backfilling — an admin correcting history (e.g. logging a check-in that happened over the phone without the app open) sets an actual date/time in the past; the target-status side effect still applies at the moment of saving, not retroactively.

Mobile vs. desktop

Same dialog on both; no meaningful layout divergence beyond the app's general responsive treatment.
