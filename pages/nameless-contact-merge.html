Nameless-Contact Name Prompt & Merge Dialog

What happens the moment anyone opens a Contact that was auto-created from an inbound call or text and never got a name — prompt first, then check for a duplicate.

Purpose

Per Role Workflows — New or unmatched callers and API Structure — Contact matching: an unmatched inbound number auto-creates a bare Contact (phone only, no name), performed by the Twilio Function. This dialog is where that gets resolved into a real, named record — and, critically, where it gets caught before becoming a duplicate of someone already in the system.

Data shown

FieldSourceNotes
Phone numberContact.phoneThe only identifying information available so far.
Name inputUser-entered
Similar existing contactsContact, fuzzy-matched against the entered nameSurfaced only after a name is typed — there's nothing to match against beforehand.

Actions

ActionGatingBehavior
Save name, no mergeedit_owner_contactSets Contact.name; the record stays distinct. Used when no similar contact is shown, or none of the ones shown are actually the same person.
Confirm a mergeedit_owner_contactSets this Contact's merged_into to the selected existing Contact. The nameless record's phone number stays associated with it (so future calls from that number still match it directly), but any display of this Contact resolves through merged_into to show the canonical person instead.
Dismiss / decide laterNoneLeaves the Contact nameless; the same prompt reappears next time anyone opens it.

States & edge cases

  • No similar contacts found — the dialog skips straight to a plain name-save with no merge step.
  • Multiple similar matches — presented as a list to choose from, with an explicit "none of these" option rather than forcing a pick.
  • No edit_owner_contact — the contact can still be viewed (per Owner/Contact detail's gating) but this prompt has nothing actionable to offer; it can show read-only ("no name on file yet") rather than the interactive form.
  • A second inbound call from the same nameless number before it's resolved — matches the existing bare Contact by phone as usual; no duplicate nameless record is created for the same number.

Mobile vs. desktop

Mobile: full-screen prompt, since it interrupts a task (usually mid-call, via Active Call Panel). Desktop: a smaller modal, since there's more surrounding context already visible.
