Assets · Page Specification

Asset List

Every tracked piece of equipment — vehicles, jump packs, anything with a maintenance cycle or a checkout log.

Purpose

Reached from the Assets top-level nav item. Viewing has no dedicated permission — same pattern as Tickets, Reservations, and Boats — only creating/editing requires manage_assets, per Permissions. Static infrastructure with no maintenance cycle and no checkout (a fixed gate, a pedestal) is deliberately not modeled as an Asset at all, per Terminology, so this list is shorter than "everything physical at the marina."

Data shown

FieldSourceNotes
Name, categoryAsset.name / categoryCategory is an admin-defined grouping.
Current statusAsset.current_statusDenormalized latest value from AssetStatusLog.
Meter / hoursAsset.meter_reading (if has_meter) or accumulated hoursA meterless asset accrues hours automatically over time rather than showing a reading, per Data Model.
Checked out?Open AssetCheckout (time_in null)Only shown for checkoutable assets.
Reservable?Asset.reservation_enabledSmall indicator; links into Reservation calendar/list filtered to this asset when applicable.

Actions

ActionGatingBehavior
Open an assetNoneNavigates to Asset detail.
Add an assetmanage_assetsCategory and maintenance-rule setup happens in Admin → Asset categories & maintenance rules; this is the day-to-day asset record itself.
Filter by category / status / checked-outNoneClient-side filter over the subscribed query.

States & edge cases

  • No manage_assets — the list is fully visible; only creation and the checkout/status/meter actions on Asset detail are hidden.
  • Meterless asset — shows accumulated hours instead of a reading, with no manual "set reading" action available for it beyond adding hours (see Meter update dialog).
  • Empty list — plain empty state.

Mobile vs. desktop

Mobile: single filterable list. Desktop: sortable table with status/checkout columns visible without opening each asset.
