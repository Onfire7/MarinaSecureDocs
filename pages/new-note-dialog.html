New Note Dialog

A freeform log entry attached to exactly one Location, Checkpoint, Boat, Contact, or Asset — no status, no priority, no workflow.

Purpose

Reached from nearly every detail page in the app (pre-attached to that record) or, less often, opened generically and completed via the Attachment-target picker. Unrestricted to create, per Permissions — Notes carry no permission gate of their own, only their attachment target's own visibility gating applies when the note is later viewed.

Data shown / form fields

FieldTypeNotes
BodyText, requiredFreeform — no markdown/rich-text editor, unlike Incident details or Ticket descriptions.
Attachment targetLocation / Checkpoint / Boat / Contact / AssetPre-filled when opened from a specific record; otherwise resolved via Attachment-target picker. Required — there's no such thing as an unattached Note, per Data Model.

Actions

ActionGatingBehavior
SaveUnrestrictedCreates the Note with author = current user, created_at = now. Appears in the Activity Log, gated there by whatever permission governs the attachment target's entity type (see Activity Log feed).
CancelNoneCloses without creating a note.

States & edge cases

  • No attachment target selected — save stays disabled; this is the one hard requirement on the dialog.
  • Offline — writes locally like any other InstantDB record.
  • Editing after save — not offered; a Note is a simple append-only log entry, with no edit action modeled anywhere in the app.

Mobile vs. desktop

Same short form on both; no meaningful layout divergence beyond the app's general responsive treatment.
