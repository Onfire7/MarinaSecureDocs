Asset Categories & Maintenance Rules

Configure the groupings assets fall into, and the rules that auto-generate maintenance tickets from meter or time thresholds.

Purpose

Reached from Admin Home, gated by manage_assets. Complements Asset detail, which shows a given asset's rules read-only and lets that same permission holder log status/checkout/meter activity day to day — this screen is specifically where the rules themselves get authored.

Data shown

FieldSourceNotes
CategoriesDistinct values of Asset.categoryAdmin-defined groupings; not a separate entity, just the set of strings currently in use plus the ability to add new ones for future assets.
Maintenance rulesAsset.maintenance_rules, per assete.g. "every 3,000 miles" or "every 90 days since last completed."

Actions

ActionGatingBehavior
Add / rename a categorymanage_assetsRenaming updates every asset currently using that category label.
Add a maintenance rule to an assetmanage_assetsMeter-based (needs has_meter = true) or time-based; a meterless asset can only take time-based rules.
Edit / remove a rulemanage_assets

States & edge cases

  • Meter-based rule on a meterless asset — not offered as an option; the form only shows rule types the asset's has_meter flag actually supports.
  • Rule triggers immediately on save — if the asset's current meter/elapsed-time already exceeds the new rule's threshold, the next scheduled evaluation (per API Structure — scheduled: maintenance-ticket-generation) generates a ticket right away rather than waiting for the next meter reading.
  • Category with no assets — allowed to exist (created ahead of assets that will use it) or to persist after its last asset is recategorized; not automatically cleaned up.

Mobile vs. desktop

Same form-and-list layout on both; no meaningful layout divergence beyond the app's general responsive treatment.
