New SMS Dialog

Start a text conversation with a contact or number that doesn't already have a thread — or jump straight to the existing one if it does.

Purpose

Reached from Comms Home. Since an SMSThread is identified by contact + line (per Data Model), this dialog's real job is picking who and which line, then handing off to that thread rather than ever creating a second thread for the same pairing.

Data shown / form fields

FieldTypeNotes
ContactSearch/select, or manual number entry
LineSelect, from MarinaSettings.phone_lines
MessageText, with the same template picker as SMS threadComposed here so the first message can go out immediately on confirming the contact/line.

Actions

ActionGatingBehavior
Sendplace_callsCalls the Twilio Function's POST /sms/send; the Function creates or appends to the matching SMSThread and creates the outbound SMSMessage. Routes into SMS thread on success.
CancelNoneCloses without sending.

States & edge cases

  • Contact/line pair already has a thread — sending routes into that existing thread rather than anything appearing duplicated; there is no way to intentionally create a second thread for the same pairing.
  • Offline — disabled the same way as SMS thread's composer, since sending requires Twilio connectivity.
  • No place_calls — this dialog is unreachable; every entry point to it is gated the same way.

Mobile vs. desktop

Same form-style dialog on both; no meaningful layout divergence beyond the app's general responsive treatment.
