Checklist Templates

Build and manage the reusable templates that generate real Checklists — items, triggers, and assignment, for Global and Role-restricted templates.

Purpose

Reached from Admin Home, gated by manage_checklists for Global and Role-restricted templates. Personal templates are a deliberate exception — per Permissions, they're always editable by their own creator regardless of this permission, so this screen also surfaces a user's own Personal templates even without manage_checklists.

Data shown

FieldSourceNotes
Name, visibilityChecklistTemplate.name / visibilityGlobal / Role-restricted / Personal.
TriggerChecklistTemplate.trigger_type / trigger_configClock In/Out, Scheduled Time, Checkpoint Visit, Incident Type, or Manual — each with its own config shape (e.g. a schedule expression, or a checkpoint + time-of-day/season condition).
AssignmentChecklistTemplate.assignment_modeTo the triggering user, or to a specified Role.
ItemsChecklistTemplate.items → ChecklistTemplateItem, orderedEach item's type (one of the 5), label, and type-specific config.

Actions

ActionGatingBehavior
Create / edit a Global or Role-restricted templatemanage_checklists
Create / edit one's own Personal templateUnrestricted for the creator, regardless of manage_checklistsNot editable by anyone else, including manage_checklists holders.
Add / reorder / remove an itemSame as editing the templateItem type is fixed once added within this builder session but can be deleted and re-added as a different type.
Configure a Door Check's expected state, a Verify Task's retry-prompt flag, a Location-Based Check's nested checklist, or a Meter Reading's asset selectionSame as editing the templateType-specific config, per Data Model — ChecklistTemplateItem.

States & edge cases

  • Editing a Personal template as its non-creator — never offered, even to a manage_checklists holder; the permission's own description explicitly carves this out.
  • Deleting a template with in-progress Checklist instances — existing instances are unaffected (a Checklist references its originating template but doesn't require the template to still exist for the instance's own data to remain valid); only future triggers stop generating new instances.
  • Scheduled-time trigger with an unusual schedule — evaluated by the checklist-triggers Netlify Scheduled Function, per API Structure; this screen just authors the trigger_config, it doesn't run it.

Mobile vs. desktop

Desktop is the primary surface for building a template's item sequence, given the amount of configuration per item. Mobile supports viewing and light edits, but authoring a new multi-item template is a desk task.
