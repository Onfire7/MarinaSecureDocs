Page Specifications

A detailed spec for every screen in the application: purpose, data displayed, actions available, permission gating, and mobile/desktop behavior, grouped to match top-level navigation.

Status. Access, Home, Checklists & Tours, Locations, Comms, Tickets & Incidents, and Reservations & Boats are written and linked below. The remaining groups are still the proposed scope — flag anything that should be added, split differently, or dropped before those are written.

Page list

Access

  • Sign in
  • User switch (shared shift devices — fast handoff without full sign-out; see Architecture)

Home

  • Dashboard (role-adaptive: shift start/end for security, ticket/incident summary, missed call badge, upcoming reservations)

Checklists & Tours

  • Checklist list (mine / assigned to my role)
  • Active checklist (in-progress, item-by-item — includes the Verify Task rejection and Door Check mismatch flows)
  • Checkpoint check-in screen (the page a scanned NFC/QR URL opens)
  • Manual check-in dialog (reason required)
  • Tour progress view (security-only — linear/freeform/randomized status)
  • Checklist detail (completed, read-only)

Locations

  • Location list / map view
  • Location detail (status, current boat, checkpoints, notes, incidents, tickets)
  • Checkpoint detail (within a location)

Comms

  • Comms home (recent calls, texts, chat rooms)
  • Active call panel/pop-up
  • Missed calls / unread SMS detail view
  • New call dialog (dialer / contact picker — place_calls)
  • SMS thread (incl. template picker)
  • New SMS dialog
  • Chat room
  • New chat room / invite screen

Tickets

  • Ticket queue (priority-sorted)
  • Ticket detail
  • New ticket form

Incidents

  • Incident list
  • Incident detail (with comments/addendums, linked tickets)
  • New incident form

Reservations

  • Reservation calendar/list — includes the map view: a list/calendar/map toggle, where the map renders each reservable Location's plotted rectangle (see MarinaMap/LocationMapPlacement in Data Model) color-coded by reservation status and labeled with the location's name
  • Reservation detail
  • New reservation form
  • Reservation check-in / check-out dialog (sets target status per its configuration)
Map-plotting data model resolved; Admin UI still to write. MarinaMap (a marina-wide master map, plus optional per-parent-Location detail maps) and LocationMapPlacement (a join entity, since one Location can be plotted on multiple maps at different coordinates) now back both this screen's map view and the still-unwritten schematic map mode on Location list / map view — one set of plotted rectangles, two different color overlays (reservation status here, occupancy/type there). Reservation eligibility is unrelated to any of this: it's the existing per-Location reservation_enabled toggle, which is what makes a mixed-use dock work (some Slips reservable short-term, others long-term leased, side by side — see Data Model — Location). Still to write: the upload + plotting UI in Admin's Location Types & Locations setup (below).

Boats

  • Boat list
  • Boat detail (owners, authorized users, current slip, lease link)

Owners & Contacts

  • Owner/Contact list
  • Owner/Contact detail
  • Nameless-contact name prompt & merge dialog
  • Lease detail (variances, documents, comments)

Assets

  • Asset list
  • Asset detail (status log, meter, maintenance rules, checkout)
  • Checkout / return dialog
  • Meter update dialog (set reading, or add hours for meterless assets)

Shared dialogs

  • New note dialog (attach to location / checkpoint / boat / contact / asset)
  • Attachment-target picker (shared by notes, incidents, tickets)

Activity Log

  • Activity Log (filterable chronological feed, permission-scoped)

Reports

  • Reports home
  • Shift report (view/resend)

Admin

  • Admin home / section picker
  • Users
  • Roles & Permissions (trinary matrix editor)
  • Checklist Templates (builder)
  • Location Types & Locations setup (incl. checkpoint setup: GUID URL, NFC/QR, GPS radius; will also need marina map upload + plotting each Location as a rectangle on it, per the Reservations map-view note above)
  • Tours setup
  • Incident Types
  • Asset categories & maintenance rules
  • SMS Templates (global; personal templates managed from Comms)
  • Marina Settings (GPS default, retention, recording/transcription, phone lines, shift report recipients)
