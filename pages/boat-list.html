Boats · Page Specification

Boat List

Every boat kept at the marina, searchable by name, owner, or slip.

Purpose

Reached from the Boats top-level nav item. Boat visibility itself is unrestricted, same as tickets and reservations; what's gated is the owner/contact detail a boat links to, per Permissions.

Data shown

FieldSourceNotes
Name, make/model, lengthBoat.name / make / model / length
Current slipBoat.current_slipNullable — a boat isn't required to have a current slip (e.g. hauled out, or newly added before assignment).
OwnerBoat.owners[0] (first in succession order)Name only requires view_owner; omitted entirely without it.

Actions

ActionGatingBehavior
Open a boatNoneNavigates to Boat detail.
Add a boatedit_owner_contactBoats are owner/contact-adjacent records, so creation reuses this permission rather than introducing a dedicated one, per the avoid-permission-bloat principle established for Dashboard cards.
Search / filterNoneBy name, owner (if view_owner), or slip.

States & edge cases

  • No view_owner — the owner column is omitted entirely from the list, not shown blank.
  • Boat with no current slip — shown plainly with an empty slip field, not hidden from the list.
  • Multiple owners — list view shows only the first (succession order); full ownership order lives on Boat detail.

Mobile vs. desktop

Mobile: single searchable list. Desktop: sortable table with slip and owner columns visible without opening each boat.
