Incidents · Page Specification

New Incident Form

Log an incident on the move — short title required, everything else optional, gated by create_incidents.

Purpose

Reached from Incident list, a checkpoint check-in, or mid-checklist. Deliberately lightweight — per Role Workflows, a guard needs to be able to log something fast, on a dark dock, without a long form standing between them and getting back to the round.

Data shown / form fields

FieldTypeNotes
TitleText, requiredShort by design.
TypeSelect, from IncidentType — expandableA new type can be added inline if none fit; the list grows per marina rather than being fixed.
DetailsRich text (markdown), optionalLightweight WYSIWYG.
Attachment targetLocation / Checkpoint / Boat / Contact / AssetPre-filled when opened from a specific context (e.g. the current checkpoint); otherwise the shared Attachment-target picker.
AssigneeSelect, optional

Actions

ActionGatingBehavior
Submitcreate_incidentsCreates the Incident with author = current user, default status Open. Routes to Incident detail on success.
Add a new incident type inlinecreate_incidents (same gate — no separate permission for expanding the type list)Creates an IncidentType record on the fly, available to all future incidents at this marina.
Raise a linked ticket immediatelyNone — ticket creation is unrestrictedOptional shortcut to also open New ticket form pre-linked, without leaving this flow first.
CancelNoneCloses without creating an incident.

States & edge cases

  • No create_incidents — this form (and every entry point into it) is unreachable; there is no way to log an incident without the permission.
  • Offline — writes locally like any other InstantDB record; fully available on a dark dock with no signal, per Architecture.
  • No attachment target available — an attachment target is required, same rule as tickets; there's no unattached incident.

Mobile vs. desktop

Mobile: a short, single-column form optimized for speed — this is the field-use case the screen is built for. Desktop: the same fields, with more room for the rich-text details editor.
