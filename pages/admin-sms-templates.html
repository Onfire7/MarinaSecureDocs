SMS Templates

Marina-wide reply templates for common texts — Global scope only; Personal templates are managed from Comms itself, not here.

Purpose

Reached from Admin Home, gated by manage_marina_settings — reused as the closest fit, since there's no dedicated template-management permission in the fixed catalog and this is fundamentally marina-wide configuration content. Personal-scope templates are each user's own and are managed inline from SMS thread, unaffected by this permission.

Data shown

FieldSourceNotes
Label, bodySMSTemplate.label / bodyGlobal scope only on this screen — SMSTemplate.scope = Global.

Actions

ActionGatingBehavior
Create / edit / delete a Global templatemanage_marina_settingsImmediately available to every user's template picker in SMS thread and New SMS dialog.

States & edge cases

  • No Global templates yet — the SMS composer's template picker simply has nothing to offer beyond any Personal templates the individual user has made.
  • Deleting a template mid-use — doesn't retroactively change any already-sent message; those are plain SMSMessage text, not a live reference to the template.

Mobile vs. desktop

Same simple list-and-form layout on both; no meaningful layout divergence beyond the app's general responsive treatment.
