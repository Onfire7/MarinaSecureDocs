Admin · Page Specification

Admin Home / Section Picker

The gateway into every configuration screen — visible in nav only to a user holding at least one manage_* permission, showing only the sub-sections they can actually open.

Purpose

Per Permissions — Admin visibility: the Admin nav item itself appears for anyone holding any manage_* permission, but each sub-section listed here is independently gated by its own corresponding permission — a user might see Admin at all only because they hold manage_assets, and find every other tile grayed out or absent.

Sub-sections, by permission

TileRequires
Usersmanage_users
Roles & Permissionsmanage_roles
Checklist Templatesmanage_checklists
Location Types & Locationsmanage_locations
Toursmanage_locations — reused; a Tour is fundamentally a checkpoint composition, and checkpoints already fall under this permission
Incident Typescreate_incidents — reused; the same permission already lets a user add a new type inline from New incident form
Asset Categories & Maintenance Rulesmanage_assets
SMS Templatesmanage_marina_settings — reused, as the closest fit; there's no dedicated template-management permission in the fixed catalog

Actions

ActionGatingBehavior
Open a tileThat tile's own permissionNavigates to the relevant sub-section.

States & edge cases

  • Holds exactly one manage_* permission — Admin appears in nav with exactly one usable tile; the rest are simply absent, not shown disabled.
  • Holds none — Admin doesn't appear in nav at all, and this page is unreachable.

Mobile vs. desktop

Mobile: tiles stack as a vertical list. Desktop: a grid, similar to the Dashboard's card layout.
