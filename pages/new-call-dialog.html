New Call Dialog

Dialer and contact picker for initiating an outbound call, gated by place_calls.

Purpose

Reached from Comms Home, Missed Comms Detail ("call back"), or a Contact/Boat/Location detail page's call action. The dialog itself is thin — it collects who to call and from which line, then hands off to the Twilio Functions API to actually place the call.

Data shown / form fields

FieldTypeNotes
ContactSearch/select, or manual number entryPre-filled when opened from a specific Contact/call-back context.
LineSelect, from MarinaSettings.phone_linesWhich marina number to call from; defaults to the single line if the marina only has one.

Actions

ActionGatingBehavior
Callplace_callsCalls the Twilio Function's POST /calls/initiate (see API Structure), which bridges the call to the requesting user's device and creates the outbound Call record itself. On success, Active Call Panel takes over.
CancelNoneCloses without placing a call.

States & edge cases

  • No place_calls — this dialog is never reachable; every entry point that would open it is itself gated the same way.
  • Contact has multiple numbers — presented as a picker rather than guessing which to dial.
  • Offline — calling requires connectivity (telephony is online-only, per Architecture); the dialog reflects this honestly (disabled state with an explanation) rather than accepting input it can't act on.
  • Only one line configured — the line selector collapses to a label rather than a meaningful choice.

Mobile vs. desktop

Mobile: a numeric keypad is available alongside the contact picker, matching phone-app conventions. Desktop: contact search and line selection are the primary interaction, with the keypad present but secondary, since desktop users are more likely dialing a known contact than a raw number.
