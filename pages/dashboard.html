Dashboard

The Home landing screen — role-adaptive, showing only what's relevant to the roles the signed-in user actually holds.

Purpose

There's no single "home" layout — the dashboard is a set of independent cards, each gated by role/permission, that assemble differently per user. A user holding both Security and Office sees both the shift card and the comms card; a pure Maintenance user sees neither. See Role Workflows for the narrative behind each card.

Cards, by relevance

CardShown whenContent
ShiftAny user (primarily Security)Start Shift button if no active Shift; while active, elapsed time, current tour progress summary, and either an End Shift button or a prompt to complete the end-of-shift checklist (see Actions below).

Actions

ActionGatingBehavior
Start ShiftNone — available to any userCreates a Shift record (guard = current user, started_at = now). May itself be configured as a checklist trigger (Clock In) — if so, the triggered checklist opens immediately.
End Shift (manual button)None; only shown when the marina has not configured an end-of-shift checklistSets Shift.ended_at, then calls the shift-report Netlify Function on-demand (see Architecture — Shift report delivery).
Complete end-of-shift checklistNone; shown instead of the manual button when a template is configuredOpens the Active checklist flow; submitting it sets Shift.ended_at and triggers the report the same way.

States & edge cases

  • No roles with any dashboard card — shouldn't occur in practice (every role maps to at least one card), but the fallback is a minimal card showing only account info and nav.
  • Shift ended offline — the dashboard reflects ended_at immediately (local write); the report itself sends once connectivity returns and the scheduled backstop sweep picks it up, per Architecture.
  • Multiple active tours/checklists — the shift card summarizes only; full detail lives on Checklist list and Tour progress.

Mobile vs. desktop

Mobile: cards stack vertically, ordered by relevance to the user's primary role, with the Shift card pinned to the top for Security. Desktop: cards arrange in a grid so several are visible without scrolling, and the layout leans toward an efficient overview rather than a single prioritized stack — see Architecture — Frontend for the general mobile/desktop divergence principle.
