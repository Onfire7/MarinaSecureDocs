Wireframes

Low-fidelity HTML wireframes for every page listed in Page Specifications, sharing a single stylesheet (wireframes/shared/styles.css) so every screen is visually consistent without any of them being a finished design.

Status. Every group is written and linked below except Admin. Flag anything that should change before it's written.

Conventions

  • Two device frames per screen — a 375px mobile frame and a ~940px desktop frame (standing in for 1280px), shown side by side wherever the two layouts genuinely diverge. Where the page spec says a screen renders identically at any width (most dialogs), a single frame stands in for both, noted explicitly.
  • Real content, not lorem ipsum — field labels and sample values are drawn from the Data Model (e.g. "Slip 14," "Jump Pack A"), not filler text.
  • Grayscale only, one exception — no brand color or type; three muted functional colors (green/amber/red) are used solely for status semantics (ticket priority, reservation status, GPS validation), never decoratively.
  • 🔒 permission tags — a dashed pill directly on a wireframe element marks it as conditionally visible/enabled, naming the permission that gates it (e.g. view_incidents) — the same gates documented in the corresponding page spec, shown in place rather than only in prose.
  • Dashed annotation boxes — italic call-outs beside non-obvious elements, explaining behavior a static mockup can't otherwise show (an async state, a disabled control, a redirect).
  • Dialogs render as a floating card over a dimmed backdrop rather than a full frame, to distinguish them from full-page screens at a glance.
