Wireframes

Low-fidelity HTML wireframes for every page listed in Page Specifications, sharing a single stylesheet (wireframes/shared/styles.css) so every screen is visually consistent without any of them being a finished design.

Sequenced after page specs. Wireframes are most useful once the page specs they illustrate are settled — building them in parallel risks redoing layout work as specs change. Once Page Specifications is reviewed and confirmed, wireframes will follow the same structure: one HTML file per screen, plus mobile and desktop states where navigation patterns diverge (see Architecture).

What these will (and won't) show

  • Layout, information hierarchy, and navigation structure for each screen, at both mobile and desktop breakpoints where they differ meaningfully.
  • Placeholder content that reflects real field names and data shapes from the Data Model — not lorem ipsum.
  • Deliberately not final visual design — no brand styling, illustration, or polish. That's a separate design pass once the underlying structure is agreed on.
